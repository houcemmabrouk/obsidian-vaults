# ⚙️ About Cognitive OS

## Pourquoi ce vault existe

Le **Cognitive OS** est mon *système d'exploitation* pour la pensée : un endroit unique pour **capturer**, **organiser**, **connecter** et **retrouver** l'information — et surtout la **transformer en production** (décisions, écrits, projets).

Il repose sur un principe : *la mémoire externe libère la mémoire de travail*. Voir [[Cognitive Load Theory]].

## Les deux moitiés du système

### 1. PARA — l'organisation orientée action
Méthode de Tiago Forte. On classe **par actionnabilité**, pas par sujet :

- **Projects** — un résultat précis + une échéance (ex. *Réussir le CFA Level 1*).
- **Areas** — une responsabilité continue à maintenir (ex. *Finance*, *Santé*).
- **Resources** — un thème d'intérêt, utile un jour (ex. *IA*, *Design*).
- **Archive** — tout ce qui est inactif.

Détails : [[PARA Method]].

### 2. Zettelkasten — la pensée en réseau
Méthode de Niklas Luhmann. Des **notes atomiques** (une idée = une note), reformulées avec mes propres mots, **densément reliées** entre elles. C'est là que naissent les idées nouvelles.

- **Fleeting** — capture brute, éphémère.
- **Literature** — notes de lecture (source + idée + réf.).
- **Permanent** — idées atomiques durables, dans mon langage, liées.

Détails : [[Zettelkasten Method]] · [[Second Brain]].

## Comment les deux cohabitent

```
Capture (Inbox / Fleeting)
        │
        ▼
Traiter → est-ce ACTIONNABLE ?
        │
   ┌────┴─────┐
  OUI        NON
   │          │
  PARA     Zettelkasten
(Projects/  (Literature →
 Areas/      Permanent)
 Resources)
```

- **PARA** répond à *« Sur quoi dois-je agir ? »*
- **Zettelkasten** répond à *« Que sais-je et comment est-ce relié ? »*

Les **MOCs** ([[07 - MOCs/🗺️ Master Index|Master Index]]) sont les cartes qui relient tout ça.

## Principes directeurs

1. **Capturer sans friction** — l'Inbox absorbe tout, on trie plus tard.
2. **Une note = une idée** (atomicité).
3. **Toujours lier** — une note orpheline est une note perdue.
4. **Reformuler, pas copier** — écrire avec ses mots (voir [[Active Recall]]).
5. **Réviser régulièrement** — weekly review + [[Spaced Repetition]].
6. **La structure émerge** — pas de sur-ingénierie des dossiers.
