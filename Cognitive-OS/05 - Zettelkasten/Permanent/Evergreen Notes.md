# Evergreen Notes

**Notes evergreen** (Andy Matuschak) : des notes conçues pour **évoluer, se cumuler et se relier** dans le temps — l'opposé de notes jetables prises pour un cours ou un projet unique.

## Les principes
- **Atomiques** — une idée par note ([[Zettelkasten Method]]).
- **Orientées concept** — titrées par l'idée, pas par la source ni la date.
- **Densément liées** — la valeur naît des connexions.
- **Écrites pour soi-futur** — formulées pour être réutilisées hors contexte.

## Le titre comme API
Un bon titre d'evergreen est une **affirmation** ou un concept clair (« [[Active Recall]] », « [[Desirable Difficulty]] »). On peut alors l'invoquer comme une fonction dans d'autres notes.

## Différence avec les notes classiques
| Notes classiques | Evergreen |
| --- | --- |
| Liées à un cours/projet | Autonomes |
| Écrites une fois | Révisées en continu |
| Rangées en dossiers | Reliées en réseau |
